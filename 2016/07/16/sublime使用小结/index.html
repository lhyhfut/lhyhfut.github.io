sublime使用小结

发表于 2016-07-16   |   分类于 Technology

下载,安装

sublime官网

插件

插件集合

1.package control

(1)通过快捷键 ctrl+` 或者菜单（View > Show Console）
打开控制台，然后粘贴下列的 Python 代码

import urllib.request,os,hashlib; h = ‘7183a2d3e96f11eeadd761d777e62404’ + ‘e330c659d4bb41d3bdf022e94cab3cd0’; pf = ‘Package Control.sublime-package’; ipp = sublime.installed_packages_path(); urllib.request.install_opener( urllib.request.build_opener( urllib.request.ProxyHandler()) ); by = urllib.request.urlopen( ‘http://sublime.wbond.net/‘ + pf.replace(‘ ‘, ‘%20’)).read(); dh = hashlib.sha256(by).hexdigest(); print(‘Error validating download (got %s instead of %s), please try manual install’ % (dh, h)) if dh != h else open(os.path.join( ipp, pf), ‘wb’ ).write(by)

另外提供Sublime text 2的Package Control的安装代码:

import urllib2,os,hashlib; h = ‘7183a2d3e96f11eeadd761d777e62404’ + ‘e330c659d4bb41d3bdf022e94cab3cd0’; pf = ‘Package Control.sublime-package’; ipp = sublime.installed_packages_path(); os.makedirs( ipp ) if not os.path.exists(ipp) else None; urllib2.install_opener( urllib2.build_opener( urllib2.ProxyHandler()) ); by = urllib2.urlopen( ‘http://sublime.wbond.net/‘ + pf.replace(‘ ‘, ‘%20’)).read(); dh = hashlib.sha256(by).hexdigest(); open( os.path.join( ipp, pf), ‘wb’ ).write(by) if dh == h else None; print(‘Error validating download (got %s instead of %s), please try manual install’ % (dh, h) if dh != h else ‘Please restart Sublime Text to finish installation’)

(2)手动安装

a.点击Preferences > Browse Packages

b.上面会打开一个文件夹,在其中找到Installed Packages文件夹(如果没找到,在父击级目录中找)

c.在插件集合上下载Control.sublime-package,然后拷贝到Installed Packages文件夹

d.重启sublime

(3)如果在preferences-package settings中看到package control这一项,则表明安装成功

注:
(1)有时候ctrl+` 打不开控制台,可能会与其他的快捷键冲突,这是可以在Preferences -> Key Bindings – Default中修改:

{ “keys”: [“ctrl+backquote”], “command”: “show_panel”, “args”: {“panel”: “console”, “toggle”: true} }

把ctrl+backquote修改为其他的快捷键就行了。

(2)Installed Packages和Packages文件夹

只有单一文件（插件名.sublime-package）的插件，被放在了Installed Packages目录中，包含有多个文件的安装包插件（比如ConvertToUTF8，IMESupport），被放在了Packages 目录中。

2.使用package control安装插件

ctrl + shift + p调出命令面板

输入install调出install package回车

输入插件。

注:

(1)已经安装的插件是不会显示在install package列表中的,但会出现在remove package列表中。

(2)除了可以在install package列表中搜索,也可以在插件集合上搜索。

(3)输入remove package则可以卸载package

(4)关闭自动更新提示

在preferences-settings User中最后添加"update_check": false

3.Theme – Soda

在Installed Packages文件夹中有Theme - Soda.sublime-package,在preferences-settings User里面添加”theme”:” Soda Dark.sublime-theme”

4.Emmet

安装后,新建一个html文件，输入html:5，tab键,会得到格式化的模板。

5.HTML-CSS-JS Prettify

安装之后在preferences-package settings-HTML-CSS-JS Prettify-set-node-path中设置你的node路径。

windows及MAC中都是用Nvm安装的。

windows中可以设置为C:/Program Files/nodejs/node.exe或C:/Users/liuhongyu/AppData/Roaming/nvm/v6.0.0/node.exe

mac中为:

/Users/liuhongyu/.nvm/versions/node/v6.0.0/bin/node

注: 为斜杠而非反斜杠。

6.ColorPicker

而在 Sublime Text 中，你可以使用内置的颜色选择器。安装完成后，只要按下Ctrl / Cmd + Shift + C 快捷键。

7.GBK to UTF8

这个在package control中没有搜到,在https://packagecontrol.io/中搜索到了`ConvertToUTF8`,再在package control中搜索ConvertToUTF8,重启，在file菜单中多了reopen with encoding及reload with encoding

8.Alignment

这里仍然有个快捷键冲突问题。默认的快捷键是super+ctrl+a，但这个与QQ的截屏快捷键冲突了。

在全局的preferences-key bindings-user中添加{ "keys": ["super+ctrl+b"], "command": "alignment" },

也可以在插件的key bindings-Default中把这个删除。

9.Sublime Terminal

这个插件可以让你在Sublime中直接使用终端打开你的项目文件夹，并支持使用快捷键。

10.JsFormat

有时从网上扒了人家的js代码来学习学习，打开发现被压缩了，这时就可以用JsFormat插件格式化js代码，恢复未压缩时候的排版，挺给力的。按快捷键Ctrl+Alt+F即可格式化当前的js文件了。

11.Ctags插件

Mac OSX下Sublime Text配置使用Ctags实现代码跳转

以前用homebrew安装过ctags,在/usr/local/bin/ctags和/usr/local/Cellar/ctags/5.8/bin/ctags

将插件下的Settings-Default中的内容全选复制到Settings-User文件中，并将其中Command的值设置为：/usr/local/bin/ctags,不行。设置为/usr/local/Cellar/ctags/5.8/bin/ctags则可以，前者只是一个快捷方式

打开工程——右键-ctags-rebuild tags,会生成ctags相关的索引目录。鼠标左键点击函数名,ctrl+t,ctrl+t就可以查找函数定义了。

在mac中测试可以,但是在win10中总是测试失败，不知道是不是ctags.exe在win10中不兼容。

12.SublimeTmpl 快速生成文件模板

ctrl+alt+h html
ctrl+alt+j javascript
ctrl+alt+c css
ctrl+alt+p php
ctrl+alt+r ruby
ctrl+alt+shift+p python

13.SublimeREPL

这可能是对程序员很有用的插件。SublimeREPL 允许你在 Sublime Text 中运行各种语言（NodeJS ， Python，Ruby， Scala 和 Haskell 等等）。

这只适合通过官方方式或者homebrew等方式安装的软件。因为SublimeREPL只能照到usr/bin及usr/local/bin。由于node是用Nvm安装的,找不到相应的node可执行文件。需要指定node目录。这在上面设置HTML-CSS-JS Prettify插件时已经说明过。python等是通过官方方式安装的,所以安装SublimeREPL之后就可以通过ctrl + b来运行python文件。但node不行。需要专门指定其可执行文件的路径。

14.编译node

Tools -> Build System -> New Build System,保存为Node.sublime-build,它会被保存在/Users/liuhongyu/Library/Application\ Support/Sublime\ Text\ 3/Packages/User里。

"cmd": ["/Users/liuhongyu/.nvm/versions/node/v4.0.0/bin/node", "--use-strict", "--harmony", "$file"], "selector": "source.js",

就可以了

.bash_profile中环境变量的设置。

export PATH=”/Users/liuhongyu/.nvm/versions/node/v4.0.0/bin”:$PATH

不能带” “

export PATH=/Users/liuhongyu/.nvm/versions/node/v4.0.0/bin:$PATH

是可以的。

15.DocBlockr

安装之后在代码段前面输入/** 然后再输入TAB/Enter键就能够得到响应的注释。

16.InsertDate

17.jsFormat

格式化js代码。比如有时候下载的是压缩的JS文件如jquery.min.js。用jsFormat格式化之后就能够以正常格式阅读代码了。

18.Terminal

当你想要打开在当前文件所在的目录的终端。但默认的Ctrl / Cmd + Shift + T会有快捷键冲突。删除在插件的Key Bindings – Default中的快捷键设置,在全局的Key Bindings – User中添加自定义的快捷键。

19.Git

这只是一个sublime git插件。

Git软件的安装与配置,与github的连接需要另外独立进行。这个插件只是把终端中的git命令操作转移到了ctrl + shift + p中,并且还不支持命令参数。

注:如果是手动安装Git插件到Packages目录,需要在Usr目录下手动创建Git.sublime-settings并在其中设置Git的路径。如windows下"git_command": "D:/IDE/Git/cmd/git.exe",sublime及其插件只能找到usr/bin及usr/local/bin,如果没有安装在这里,就需要另外指定路径。
