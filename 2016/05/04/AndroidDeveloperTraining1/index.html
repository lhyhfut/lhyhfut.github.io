Starting Another Activity

经过上面的学习,我们已经有了一个activity (a single screen) with a text field and a button。本节add some code to MyActivity that starts a new activity when the user clicks the Send button.也就是说另外添加一个界面,当用户点击send按钮的时候跳转到这个界面。

Respond to the Send Button

(1)修改 res/layout/content_my.xml文件。添加android:onClick 属性到 <Button>元素.

android:onClick="sendMessage"

android:onClick的值sendMessage就是你的activity里的方法名。当用户点击这个button的时候就会跳转到这个activity。

(2)修改app/src/main/java/com/example/liuhongyu/helloworld/MyActivity.java

在MyActivity class内,添加sendMessage() method。

public void sendMessage(View view){
}

注意里面的View view参数。保证了点击的是View

Build an Intent

(1)在上面的sendMessage method内创建一个Intent 来开启一个名为DisplayMessageActivity的:

Intent intent = new Intent(this, DisplayMessageActivity.class);

The reference to DisplayMessageActivity will raise an error if you’re using an IDE such as Android Studio because the class doesn’t exist yet. Ignore the error for now; you’ll create the class soon.即暂时会因为没有DisplayMessageActivity而报错,待会创建它

An Intent is an object that provides runtime binding between separate components (such as two activities). The Intent represents an app’s “intent to do something.” You can use intents for a wide variety of tasks, but most often they’re used to start another activity 即Intent常用来启动另一个activity

上面构造函数的2个参数:

A Context as its first parameter (this is used because the Activity class is a subclass of Context)

The Class of the app component to which the system should deliver the Intent (in this case, the activity that should be started)

(2)在MyActivity.java顶部import Intent class

报错:Android studio不能够resolve symbol Intent。你必须import the Intent class:

import android.content.Intent;

小技巧: In Android Studio, press Alt + Enter (option + return on Mac) to import missing classes.

(4)在MyActivity.java顶部import the EditText class

(5)Assign the text to a local message variable, and use the putExtra() method to add its text value to the intent.

(6)在MyActivity class,添加EXTRA_MESSAGE的定义

(7)在sendMessage()方法内,调用startActivity()方法,传入(1)中的indent参数。

The system receives this call and starts an instance of the Activity specified by the Intent. Now you need to create the DisplayMessageActivity class in order for this to work.

但上面也提示了可以用git add -f来手动添加这些文件

在github上新建一个仓库Create a new repo: AndroidDeveloperTraining,添加一个README

从上面找到远程仓库的SSH

终端把本地的含有.git文件见的仓库和github上的远程仓库联系起来:git remote add origin git@github.com:lhyhfut/AndroidDeveloperTraining.git

然后把本地内容推送到远程仓库: git push -u origin master

报错:Updates were rejected because the remote contains work that you do not have locally.

原因:远程仓库中有README,本地仓库没有,这样不能推送

在settings——Delete this repository中删除刚才的远程仓库。新建一个不含README的仓库。

不需要重新用git add remote origin SSH建立联系了。因为2个仓库的SSH是一样的。只需要重新git push -u origin推送就行了。

在github上看每次commit的代码修改就比较方便:

然后上传到远程仓库中:

git add *

git commit -m "solve cannot run second activity"

git push -u origin master

Supporting Different Devices

android有很多种不同屏幕尺寸的机型。这意味着你需要适配很多种。Some of the important variations that you should consider include different languages, screen sizes, and versions of the Android platform.

Supporting Different Languages

为了便于管理UI strings,android在每一个project里都有一个resources 目录。如果你用Android SDK Tools创建project。则这个tools会在项目顶层创建一个res/目录。它里面有很多子目录用于处理不同的资源类型。还有一些默认的资源文件如res/values/strings.xml,用于存储string values.

Create Locale Directories and String Files

android studio 在res/中添加了不同的values目录,用于支持更多类型的语言。里面都有各自的strings.xml

Use the String Resources

你可以通过使用resource name defined by the <string> element's name attribute在源码中引用string resources或者其他的XML files。在源码中,可以使用R.string.<string_name>类似语法来引用string resource。
