科技和文化的一些趋势

发表于 2016-06-19   |   分类于 culture

写在前面

网络上也是各种言论大谈趋势。虽然对于工程师而言还是埋头搞技术务实一些,但不能没有方向。宽广的视野也是很重要的。技术和修行上都要一门深入,但也要在正确的方向下前进。所谓正确包含两点:感兴趣,切合实际。感兴趣才会有动力,才能坚持下去。从修行上来说,兴趣也就是愿力。这是推动人前进的第一因。但大凡人类都是要受限于这个身体,时代的思想,时间和空间,所以要切合实际。当然有也有很多避世的理想主义者,他们生活的也很快乐,但不在此讨论范围之类。这次讨论的还是入世的,贴合生活的。

我的选择

这个宇宙有太多奇妙的东西值得探索,追寻,了解。但人的一生有限,不论是在时间上还是在精力,智慧,环境上都是。所以要做出选择。大学的时候已经确定,从2个方向进行探索:科技和文化。在具体一些,科技方面选择计算机与互联网,文化方面选择中国的儒道释。通往终极了解的道路有很多。每个人的业力,兴趣爱好,时代环境不同,所选择的道路也不同。

科技

科学技术是第一生产力。因为这个缘故,它比较符合”切合实际”这一条原则。所以选择它作为切入点,至少生活上也有所保障。

现在是互联网的时代。各种传统行业也都尝试着朝互联网靠近。医疗,餐饮,出行,娱乐…总之,已经渗透到人的生老病死,衣食住行,物质精神生活都有涉及。

互联网为什么会这么流行呢?连接。提供人与人的连接,人与物的连接,物与物的连接。连接多了,世界就变小了。交流起来也就容易多了。

最初是PC,门户网站,然后是移动端app,这些满足了人们获取信息以及社交的需求。当然现在是信息过剩。手机等电子设备占据了我们太多时间,很多在我看来对人是没有必要甚至有害的。

下一步会是什么呢?马省长说是大数据,我们已经由IT已经进入了DT时代.腾讯当然继续它的社交了。小米也在忙着它的生态布局。也有人说是云计算,微软,亚马逊等也都在云服务方面做了很多努力。也有人说是人工智能,google,facebook,百度也都有自己的智能布局,比如人工智能围棋啦,无人驾驶啦。也有说是物联网,Intel也在忙着收购FPGA公司,忙着在物联网时代与arm一决高下。还有特斯拉等,颠覆了人们对传统汽车行业的认知。大家也都称他为汽车界的苹果公司。

个人还是比较看好人工智能和物联网。也就是软硬结合的路线。人工智能也可以单独走软件的路子,比如社交,比如聊天机器人,也可以走硬件的路子。当然后者还是要依赖于软件。未来还是肯定是物联网的时代,肯定会有更多更细微的硬件出现。说不定到时候人的身体内部都布满了各种硬件。那如何去控制它们?如何保证它们有条不紊的运行,显然还是要靠机器,要考人工智能。大数据,云计算等也是建立在人工智能的基础上。

上面貌似离自己的生活还是有些远。对于自己的话,要想与这些方面有些联系,机械,电子(嵌入式)电路,前端,后端都需要了解。也就是全栈。现在全栈不仅仅指web开发了。懂前后端就可以了,可能还需要了解更多。那这么多到底怎么操作呢?还好javascript能够做大多数事情,所以也就选择JS全栈。其余的先放一放。有需求的时候再做。

科技方面暂时想到这里。

文化

仅仅有科技是不行的。有时候甚至觉得科技可以丢掉。古人没有什么高科技不也生活的挺好吗?但既然出生在这个时代,就用科技做一些有益的事情也无可厚非。还有就是文化。文化不仅仅指那些离我们很遥远的哲理思辨。文化渗透在人们的日常生活中,言行举止中。科技其实也算作一种理性的文化。现代社会比以前浮躁,喧嚣一些。一方面也是由于互联网的发展,有什么奇异的事情都会传播的很快,引起全民讨论。今天先不谈传统文化以及现代文化的异同及优缺点。主要谈互联网教育。

教育和互联网

教育在文化中是个重要的环节。现在是互联网时代。互联网教育也被炒的比较火。很多传统课堂都搬到了互联网上,一些明星教师的收入也很高。对学生而言以前的学习资料很有限,但现在大家都能从网上获取资料了。已经不是有没有而是如何选择资料的问题。互联网教育大概可以分为学生教育和成人教育两方面。

学生教育

学生教育包含的也很多。现在着重谈应试教育。素质教育什么的可操控性没有应试教育那么强。应试教育比较机械,但从互联网教育的角度来讲可操控性也就比较强一些。课程,作业,考试,反馈。完成这几个环节,成绩上有所进步往往就意味着教育有成果。互联网能够把教师,学生,课程三者有效的联系起来。现在互联网教育也大多数是这么做的,搭建一个平台,把资源往上放,提供一些工具就OK了。但其实还可以借鉴一下传统教育,某些方面它做的很好,并非一无是处。

传统教育除了正规的课堂教育以外,还有找家教。也就是一对一的辅导。学生在课堂上消化不了的时候往往需要家教。

互联网教育可以从家教这方面切入一下,虽然网上也可以找家教,但现在貌似还没有做的特别好的。

还有一个方面是社交。可能大家往往认为学生教育往往就是你教我学然后就没有了。但社交在这方面也还是有很大潜力的。比如可以考虑做一个学生版的知乎。专门提供问答。

暂时先写到这里。

成人教育

与学生应试教育相比,成人教育可能就算是素质教育了。学生面临的是考试,升学,成人面临的是掌握技能,工作。具有很强的实践性。比如现在的各种互联网培训课程。一种就是小规模收费的培训机构,提供IOS,Android,web方面的培训,保证就业。另外一种就像网易云课堂,百度传课这种平台性质的。本身不提供课程,搭建一个平台,让一些大牛们自己在上面讲课。如果觉得好,也可以设置收费。与前面的培训机构相比,这上面主要是个人的,可能也有一些团队专门在做这个,但专门一个公司在这个平台上做线上教育的估计就很少了。

apply plugin: ‘com.android.application’ //应用程序

//android自身的配置

android {//点击project structure可以查看

compileSdkVersion 23
buildToolsVersion "23.0.3"

defaultConfig {
    applicationId "com.example.liuhongyu.helloworld"//应用程序包名
    minSdkVersion 8
    targetSdkVersion 23
    versionCode 1
    versionName "1.0"
}
buildTypes {
    release {
        minifyEnabled false  //设置混淆
        proguardFiles getDefaultProguardFile('proguard-android.txt'), 'proguard-rules.pro' //proguard-android.txt指定混淆文件,系统指定的规则
    }
}

}

//依赖的配置

dependencies {

compile fileTree(`dir: 'libs', include: ['*.jar']`) // jar包和库都会在这里进行编译
testCompile 'junit:junit:4.12'
compile 'com.android.support:appcompat-v7:23.3.0'
compile 'com.android.support:design:23.3.0'

}

Working with System Permissions

To protect the system’s integrity and the user’s privacy, Android runs each app in a limited access sandbox. If the app wants to use resources or information outside of its sandbox, the apphas to explicitly request permission. Depending on the type of permission the app requests, the system may grant the permission automatically, or the system may ask the user to grant the permission.

Declaring Permissions

Every Android app runs in a limited-access sandbox. If an app needs to use resources or information outside of its own sandbox, the app has to request the appropriate permission. You declare that your app needs a permission by listing the permission in the App Manifest.

Determine What Permissions Your App Needs

For a list of system permissions, see Normal and Dangerous Permissions.

Add Permissions to the Manifest

To declare that your app needs a permission, put a uses-permission element in your app manifest, as a child of the top-level manifest element.

For more information about the different kinds of permissions, see Normal and Dangerous Permissions.

Requesting Permissions at Run Time

Beginning in Android 6.0 (API level 23), users grant permissions to apps while the app is running, not when they install the app.

System permissions are divided into two categories, normal and dangerous
